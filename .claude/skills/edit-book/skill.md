# 책 편집 스킬

전체 원고를 통합적으로 읽고 4축 편집 체크리스트를 적용하여 품질을 검증하는 스킬.

## 입력 파라미터

| 파라미터 | 필수 | 기본값 | 설명 |
|----------|------|--------|------|
| chapters_dir | O | `_workspace/book/chapters/` | 챕터 파일 디렉토리 |
| blueprint_path | O | `_workspace/book/blueprint/book-blueprint.md` | 책 전체 구조 경로 |
| target_chapters | X | - | 특정 챕터만 재검토 시 번호 목록 |

## 실행 절차

### 1단계: Blueprint 로드

`book-blueprint.md`를 읽어 편집 기준을 확립한다:
- 서사 아크 구조 (3막, 각 막의 역할)
- 톤 가이드 (유지할 톤, 조정할 부분)
- 챕터 목록 및 각 챕터의 핵심 논지

### 2단계: 전체 원고 순차 읽기

챕터를 번호순으로 읽는다:
- `ch-00-prologue-draft.md` (서론)
- `ch-01-draft.md` ~ `ch-{NN}-draft.md` (본문)
- `ch-{NN}-epilogue-draft.md` (결론)

`target_chapters`가 지정된 경우 해당 챕터만 읽되, 앞뒤 1개 챕터도 함께 읽어 흐름을 확인한다.

### 3단계: 4축 편집 체크리스트 적용

#### 축 1: 일관성 (Consistency)

| 항목 | 확인 방법 |
|------|----------|
| 용어 통일 | 같은 개념에 다른 용어를 사용하는 곳 표시. 예: "의사결정"과 "결정" 혼용 |
| 톤 통일 | 경어/반말, 화자 시점이 일관되는지. Blueprint 톤 가이드와 대조 |
| 인물/사례 참조 | 앞에서 소개한 사례를 뒤에서 참조할 때 세부 사항이 일치하는지 |
| 시제 일관 | 과거/현재 시제 혼용이 자연스러운 범위 내인지 |

#### 축 2: 흐름 (Flow)

| 항목 | 확인 방법 |
|------|----------|
| 챕터 간 전환 | 마무리→다음 도입이 자연스러운지. 급작스러운 주제 변경이 없는지 |
| 서사 아크 체감 | 제1막에서 문제가 제기되고, 제2막에서 탐구가 심화되며, 제3막에서 통합되는 흐름이 느껴지는지 |
| 내적 논리 | 앞 챕터의 주장과 뒤 챕터의 주장이 모순되지 않는지 |
| 페이스 | 긴 챕터와 짧은 챕터가 적절히 배분되어 독자 피로가 관리되는지 |

#### 축 3: 깊이 (Depth)

| 항목 | 확인 방법 |
|------|----------|
| 표면적 요약 방지 | 블로그의 핵심만 나열한 수준이 아닌, 더 깊은 분석이 있는지 |
| 새로운 연결 | 블로그에 없던 새로운 통찰이나 포스트 간 연결이 있는지 |
| 구체적 사례 | 추상적 주장에 구체적 사례가 뒷받침되는지 |
| 독자 가치 | 읽은 후 독자가 가져갈 수 있는 실행 가능한 통찰이 있는지 |

#### 축 4: 중복 (Redundancy)

| 항목 | 확인 방법 |
|------|----------|
| 사례 중복 | 같은 사례가 2개 이상 챕터에서 등장하지 않는지 |
| 논지 중복 | 같은 주장을 여러 챕터에서 반복하지 않는지 |
| 메타포 중복 | 같은 비유를 과도하게 재사용하지 않는지 |

### 4단계: 챕터별 등급 판정

| 등급 | 기준 | 처리 |
|------|------|------|
| **PASS** | 4축 모두 문제 없음 | draft를 final로 복사 |
| **MINOR** | 용어 통일, 전환 문장 보강, 오탈자 등 편집자가 직접 수정 가능한 수준 | 직접 수정 후 final 생성 |
| **MAJOR** | 핵심 논지 누락, 심각한 흐름 단절, 구조적 문제, 대량 중복 등 재작성 필요 | 반려 — edit-report에 상세 기록 |

### 5단계: MINOR 수정 실행

MINOR 판정 챕터에 대해 직접 수정한다:
- 용어 통일: 첫 등장 용어를 기준으로 통일
- 전환 문장: 필요한 경우 챕터 시작/끝에 1~2문장 추가
- 오탈자/문법: 직접 수정
- 수정된 결과를 `ch-{NN}-final.md`로 저장

### 6단계: Edit Report 생성

```markdown
# 편집 리포트

## 전체 요약
{전반적 평가 — 강점과 개선 사항}

## 편집 통계
| 항목 | 수 |
|------|---|
| 총 챕터 수 | {N} |
| PASS | {N} |
| MINOR | {N} |
| MAJOR | {N} |

## 챕터별 편집 결과
| # | 챕터 제목 | 등급 | 주요 수정/반려 사유 |

## MINOR 수정 상세
### 챕터 {NN}: {제목}
- 수정 1: {무엇을 어떻게 수정했는지}
- ...

## MAJOR 반려 상세
### 챕터 {NN}: {제목}
**반려 사유**: {구체적 문제}
**수정 방향**: {기대하는 수정 방향}
**참고**: {blueprint의 어떤 부분과 불일치하는지}
```

### 7단계: 통합본 생성 (MAJOR 없을 때)

모든 챕터가 PASS 또는 MINOR(수정 완료)이면 통합본을 생성한다:

```markdown
# {책 제목}

> {부제}

---

{프롤로그 내용}

---

{챕터 1 내용}

---

{챕터 2 내용}

---

...

{에필로그 내용}

---

<!-- 편집 완료: {날짜} -->
<!-- 총 챕터: {N}개 -->
<!-- 총 분량: {N}자 -->
```

파일명: `_workspace/book/manuscript.md`

MAJOR 챕터가 있으면 통합본을 생성하지 않고, edit-report만 생성한다.

## 주의사항

- 작가의 고유한 표현이나 스타일은 존중한다. 편집은 구조적 문제에 집중하고, 문체 선호에 의한 불필요한 재작성은 하지 않는다
- MAJOR 판정은 신중하게 한다. "더 좋아질 수 있는" 수준이면 MINOR로 직접 개선하고, "근본적으로 다시 써야 하는" 수준일 때만 MAJOR로 반려한다
- 전체 챕터의 50% 이상이 MAJOR이면 구조 자체에 문제가 있을 수 있다. 이 경우 edit-report에 "전체 구조 재검토 권고"를 추가한다
