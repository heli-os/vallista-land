# 블로그 콘텐츠 분석 스킬

블로그 포스트 전량을 체계적으로 분석하여 책으로 발전시킬 후보를 도출하는 스킬.

## 실행 절차

### 1단계: 포스트 인벤토리 수집

포스트 디렉토리를 탐색하여 전체 목록을 확보한다.

```
packages/blog/content/posts/*/index.md
```

각 포스트의 frontmatter에서 추출할 필드:
- `title`, `tags`, `series`, `date`, `draft`, `description`

효율적 읽기 순서:
1. 먼저 모든 포스트의 frontmatter만 수집 (각 파일 상위 30줄)
2. 태그/시리즈 기반으로 그룹핑
3. 그룹별로 본문 전체 읽기

### 2단계: 태그 기반 1차 그룹핑

허용 태그 7개 기준으로 분류한다:
- 에세이, 기술, 성장, 조직, 스타트업, 회고, 리뷰

태그가 여러 개인 포스트는 primary 태그(첫 번째)를 기준으로 하되, 모든 태그를 기록한다.

### 3단계: 시리즈 기반 연결 분석

기존 시리즈를 확인하고 시리즈 내 포스트들의 주제 연속성을 평가한다:
- "볼타 이야기", "생각이 세계가 되는 순간들", "좌뇌의 소설" 등
- 시리즈 내 포스트 수, 주제 일관성, 발전 방향

### 4단계: 의미론적 교차 주제 발견

태그가 다르더라도 서사적으로 연결되는 포스트를 찾는다:
- 같은 메타포를 사용하는 포스트
- 같은 문제를 다른 각도에서 다루는 포스트
- 경험의 시간적 연속성이 있는 포스트 (before/after)

교차 주제를 클러스터로 명명한다. 예: "판단의 구조" (인지과학 + 의사결정 + 스타트업 전략)

### 5단계: 책 후보 생성

클러스터 분석 결과를 바탕으로 3~5개 책 후보를 생성한다.

**후보 점수 기준 (4축)**:

| 축 | 기준 | 가중치 |
|----|------|--------|
| **분량 충분성** | 소스 포스트 총 단어 수 (15,000자 이상 권장) | 25% |
| **주제 깊이** | 하나의 주제를 다양한 각도에서 다루는 포스트 수 | 30% |
| **서사적 연결성** | 포스트들이 자연스러운 서사 흐름을 형성하는지 | 30% |
| **독자 가치** | 특정 대상 독자에게 명확한 가치를 제공하는지 | 15% |

각 축을 1~5점으로 평가하고, 가중 평균으로 총점을 산출한다.

### 6단계: 산출물 생성

#### content-analysis.md 포맷

```markdown
# 블로그 콘텐츠 분석

## 분석 개요
- 분석 일시: {날짜}
- 총 포스트 수: {N}개
- 총 단어 수: {N}자

## 태그별 분포
| 태그 | 포스트 수 | 총 단어 수 |

## 시리즈 현황
| 시리즈명 | 포스트 수 | 주제 일관성 |

## 전체 포스트 분류 맵
| # | 제목 | 태그 | 시리즈 | 날짜 | 단어 수 | 클러스터 | 핵심 키워드 |

## 교차 주제 클러스터
### 클러스터 1: {명칭}
- 포함 포스트: ...
- 연결 논리: ...
```

#### book-candidates.md 포맷

```markdown
# 책 후보 제안

## 후보 1: {제목}
### 부제: {부제}
### 대상 독자: {구체적 독자층}
### 소스 포스트 ({N}개)
- {포스트 제목} — {이 포스트에서 활용할 핵심 아이디어}
- ...

### 예상 구조
| 챕터 | 가제 | 관련 포스트 |

### 점수
| 축 | 점수 | 근거 |
|----|------|------|
| 분량 충분성 | {1~5} | {근거} |
| 주제 깊이 | {1~5} | {근거} |
| 서사적 연결성 | {1~5} | {근거} |
| 독자 가치 | {1~5} | {근거} |
| **총점** | **{가중평균}** | |

### 강점
- ...
### 약점
- ...

---
(후보 2, 3, ... 반복)
```

## 주의사항

- 포스트 단어 수는 한국어 기준. 공백 제외 글자 수가 아닌, 띄어쓰기 포함 단어 단위로 센다
- 레거시 포스트 중 태그가 없는 것이 있을 수 있다. 본문을 읽고 가장 적합한 태그를 추정하여 분류한다
- 후보 제안 시 기존 시리즈를 그대로 책으로 만드는 것도 가능하지만, 시리즈를 넘어서는 교차 주제 후보도 반드시 1개 이상 포함한다
