# 책 구조 설계 스킬

선택된 책 후보의 소스 포스트를 정독하여 핵심 통찰을 추출하고, 전체 서사 아크와 챕터별 상세 설계서를 생성하는 스킬.

## 입력 파라미터

| 파라미터 | 필수 | 기본값 | 설명 |
|----------|------|--------|------|
| candidate_id | O | - | 선택된 후보 번호 |
| analysis_path | O | `_workspace/book/analysis/book-candidates.md` | 분석 결과 경로 |
| user_direction | X | - | 사용자의 추가 방향성 |

## 실행 절차

### 1단계: 후보 정보 로드

`book-candidates.md`에서 선택된 후보의 정보를 읽는다:
- 소스 포스트 목록
- 예상 구조 (초안)
- 대상 독자
- 강점/약점

### 2단계: 소스 포스트 정독

후보의 소스 포스트 전체를 읽고, 각 포스트에서 핵심 통찰을 추출한다:
- 포스트별 1~3개의 핵심 아이디어
- 재사용할 만한 구체적 사례/경험
- 인상적인 메타포나 표현
- 다른 포스트와의 연결 지점

### 3단계: 서사 아크 설계

3막 구조를 기본으로 전체 서사를 설계한다:

**제1막: 문제 제기** (전체의 약 25%)
- 독자가 공감할 수 있는 문제나 질문을 제시
- 기존 통념에 대한 도전 또는 새로운 시각 제안
- 챕터 1~2개

**제2막: 탐구와 심화** (전체의 약 50%)
- 제1막에서 제기한 문제를 다양한 각도에서 탐구
- 구체적 사례, 분석, 경험을 통한 논증
- 챕터 4~8개

**제3막: 통합과 비전** (전체의 약 25%)
- 탐구 결과를 종합하여 새로운 관점이나 프레임워크 제시
- 독자에게 실행 가능한 통찰 전달
- 챕터 2~3개

### 4단계: 챕터 분할

서사 아크 위에 챕터를 배치한다:
- 총 8~15개 챕터 (서론/결론 포함)
- 챕터당 목표 분량: 3,000~5,000자
- 하나의 소스 포스트가 여러 챕터에 분산될 수 있다
- 하나의 챕터가 여러 소스 포스트를 엮을 수 있다

### 5단계: 챕터별 Spec 작성

각 챕터에 대해 설계서를 작성한다:

```markdown
# 챕터 {NN}: {제목}

## 서사 아크 내 위치
{이 챕터가 전체 서사에서 담당하는 역할. 제N막의 어디에 해당하는지}

## 핵심 논지
{이 챕터의 중심 주장 1~2문장}

## 소스 포스트
- {포스트 제목} → 사용할 부분: {구체적 섹션/아이디어}
- ...

## 구성
### 도입: {어떻게 시작할지 — 질문, 사례, 메타포 등}
### 전개 1: {첫 번째 논점과 뒷받침 사례}
### 전개 2: {두 번째 논점과 뒷받침 사례}
### 마무리: {핵심 메시지 요약 + 다음 챕터로의 전환}

## 전환 지침
- 이전 챕터({NN-1})에서 이어받을 맥락: {구체적 내용}
- 다음 챕터({NN+1})로 넘길 맥락: {구체적 내용}

## 예상 분량
{3,000~5,000자}

## 주의사항
{이 챕터 집필 시 특별히 유의할 점}
```

서론(프롤로그)과 결론(에필로그)에 대해서도 spec을 작성한다:
- `ch-00-prologue-spec.md`: 책 전체를 관통하는 질문이나 이미지로 시작
- `ch-{NN}-epilogue-spec.md`: 제3막 이후 독자에게 남기는 여운

### 6단계: 소스 매핑

포스트→챕터 매핑 테이블을 생성한다:

```markdown
# 소스 포스트 → 챕터 매핑

| 포스트 제목 | 챕터 | 사용 부분 | 활용 비중 |
|------------|------|----------|----------|
| {제목} | Ch.3, Ch.7 | 핵심 사례, 결론 | 높음 |
| {제목} | Ch.5 | 도입 메타포 | 낮음 |
| ... | ... | ... | ... |

## 미사용 포스트
| 포스트 제목 | 제외 사유 |
```

### 7단계: Blueprint 통합

`book-blueprint.md`에 전체 구조를 통합 정리한다:

```markdown
# {책 제목}
## 부제: {부제}
## 대상 독자: {구체적 독자층}

## 서사 아크
### 제1막: {문제 제기} (챕터 1~N)
{이 막에서 독자에게 전달할 핵심 경험}
### 제2막: {탐구와 심화} (챕터 N+1~M)
{이 막에서 다루는 주요 논점들}
### 제3막: {통합과 비전} (챕터 M+1~끝)
{이 막에서 도달할 결론}

## 전체 톤 가이드
{블로그의 철학적+실용적 톤을 책에 맞게 어떻게 조정할지}
- 블로그의 "~가 아닐까" 탐구적 톤 유지
- 하지만 책의 서사에 맞게 더 깊은 맥락과 논증을 추가
- 메타포는 챕터 내에서 발전시키되, 챕터 간 반복은 피함

## 챕터 목록
| # | 제목 | 핵심 논지 | 소스 포스트 수 | 예상 분량 |

## 사용하지 않는 포스트 목록
| 포스트 제목 | 제외 사유 |
```

## 주의사항

- 블로그 포스트의 시간순 배열을 그대로 책의 챕터 순서로 사용하지 않는다. 서사적 논리가 시간순보다 우선한다
- 소스가 부족한 챕터에는 spec의 "주의사항"에 "추가 집필 필요: 소스 포스트에 없는 내용이므로 writer가 직접 구성해야 함"을 명시한다
- 챕터 수가 8개 미만이면 분량이 부족할 수 있다. 15개를 초과하면 독자가 지칠 수 있다. 이 범위를 벗어나면 사유를 blueprint에 기록한다
