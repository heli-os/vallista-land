# 포스트 1차 윤문 스킬 (humanize-post)

블로그 포스트의 AI-tell 패턴을 humanize-kr v1.5 quick-rules 기준으로 검출하고, S1 결정적 패턴만 핀포인트 수정한다. 인용·고유명사·수치·작가 voice는 100% 보존.

## 글로벌 humanize-kr 와의 관계

- **글로벌 humanize-kr** (`~/.claude/skills/humanize-kr/`): 5인 파이프라인 정밀 윤문(별도 Claude 세션, 변경률 30% 한도, S2까지 처리). 장문/S1 다발 의심 시 사용.
- **본 스킬 (humanize-post)**: 인라인 1차 윤문. **S1만, 변경률 10% 이내, 같은 세션 내 Edit으로 즉시 수정**. 일상 포스트 작성 직후 자동 호출.

본 스킬 처방의 분류 체계 SSOT는 `~/.claude/skills/humanize-kr/.claude/skills/humanize-korean/references/quick-rules.md`. 호출 시 필요하면 직접 참조 가능.

## S1 패턴 — 자동 수정 대상

| ID | 패턴 | grep 시그니처 | 처방 |
|---|---|---|---|
| A-1 | "~에 대해/대한" | `에 대해\|에 대한` | 목적격 직결("X에 대해 논의" → "X를 논의") |
| A-2 | "~를 통해/통하여" 남발(3회+) | `를 통해\|을 통해\|를 통하여` | "~로", "~해서"로 분산 |
| A-3 | "~에 있어(서)" | `에 있어` | "~에서", "~을 볼 때" |
| A-7 | "가지고 있다" | `가지고 있다` | 형용사형으로 |
| A-8 | 이중 피동 "~되어진다 / ~지게 된다" | `되어진다\|지게 된다` | 능동/단일 피동 |
| C-1 | 첫째/둘째/셋째 3단 공식 | 문두 `첫째,` + `둘째,` + `셋째,` 동시 등장 | 산문 흐름으로 해체 |
| C-5 | 이모지 (장르: 칼럼/리포트/에세이) | 유니코드 이모지 | 전량 삭제 |
| C-10 | 콜론·em-dash 부제 헤딩 | `^### .+: ` 또는 `^### .+ — ` (헤딩 1개에 1회 이상) | 부제 제거, 정보는 본문 첫 문장으로 |
| D-1 | 결론·요약류 | `결론적으로\|요약하면\|정리하자면\|결론은 \|종합하면` | 삭제 또는 본문 흡수 |
| D-2 | 의의·중요성 과장 | `시사하는 바\|주목할 만\|간과할 수 없\|무시할 수 없\|의미심장` | 구체 결론으로 |
| D-3 | 본질·핵심 부사 | `본질적으로\|핵심적으로` | 삭제 |
| D-6 | 결말 공식 | `~할 때다\|~할 때입니다\|지금이야말로\|할 시점입니다` | 평서로 닫기 |
| H-1 | 문두 접속사 5회+ | 문두 `또한\|따라서\|즉,\|나아가\|아울러\|게다가\|더욱이` 누적 5회+ | 70% 제거 |
| I-1 | "~인 것이다" 종결 | `인 것이다\|한 것이다\|는 것이다` 결말 | 평서 종결 ("~다") |
| J-2 | 따옴표 강조 5회+ (인용 제외) | 비인용 `'..'`/`"..."` 카운트 5회+ | 첫 등장만 살리고 평어로 |

> 주의: A-2, J-2, H-1은 빈도 임계가 있는 패턴이라 1~2회는 자연스럽다. 임계 충족 시에만 finding.

## Do-NOT — 절대 보호 영역

탐지·치환 후보에서 **반드시 제외**:
- 큰따옴표 안 직접 인용 (`"..."`)
- 코드 펜스 (`` `...` `` / ` ```...``` `)
- 책·작품 제목 (《》, 〈〉)
- 인명·기관명·제품명·모델명
- 영어 약어(API, LLM, GPU, MCP, SDK 등 업계 표준)
- 수치·날짜·단위
- 법률·학술 정의 인용
- 작가의 핵심 비유 voice (예: 글 전체에서 의도적으로 어떤 추상 개념을 의인화하는 경우, D-5는 절반 이상 보존 권고)

## 실행 절차

### 1단계: 대상 파일 결정

호출 인자가 있으면 그 경로 사용. 없으면 다음 우선순위로 자동 결정:
1. write-essay/write-report/write-chapter/explain-paper가 직전에 생성·수정한 파일 (스킬 호출 인자로 전달)
2. `git status`에서 untracked 또는 modified 상태인 한국어 산문 마크다운 파일:
   - `packages/blog/content/posts/**/index.md` (블로그 포스트)
   - `_workspace/book/chapters/ch-*.md` (책 챕터 — frontmatter 없음)
3. 후보 0개 → 사용자에게 파일 경로 요청
4. 후보 2개+ → 사용자에게 선택 요청

대상 파일은 한국어 산문 본문이 있는 마크다운이어야 한다. 코드/README/스킬 정의 등은 대상 외.

### 2단계: 본문·Do-NOT 영역 추출

```
- frontmatter (--- 사이) 제외
- 본문에서 Do-NOT 영역(인용·코드·고유명사 등) 위치 인덱싱
- 카운트용 본문 = 전체 본문 - Do-NOT 영역
```

### 3단계: S1 패턴 검출

각 S1 패턴마다 grep으로 매칭:
- 매칭 위치가 Do-NOT 영역 안이면 제외
- 빈도 임계 패턴(A-2, J-2, H-1)은 임계 충족 시에만 finding
- finding마다 `{id, category, severity: "S1", text_span, reason, suggested_fix}` 기록

### 4단계: 윤문 결정 트리

| S1 finding 수 | 처리 |
|---|---|
| 0건 | SKIP. 결과: PASS, 등급 A. 7단계로 직행. |
| 1~5건 | 자동 핀포인트 윤문 (5단계). |
| 6건 이상 | 자동수정 **중단**. 사용자에게 글로벌 humanize-kr 정밀 윤문 권고: `cd ~/.claude/skills/humanize-kr && claude` → "이 글 자연스럽게 윤문해줘". |

### 5단계: 핀포인트 윤문 적용

각 finding마다 처방대로 Edit 호출. 수정 중 다음 게이트 모니터링:

- **변경률 게이트**: 누적 변경 글자수 / 본문 글자수 ≤ **10%**. 초과 시 즉시 중단 + 사용자에게 권고("정밀 윤문 권장").
- **Do-NOT 침범 게이트**: Edit 후 인용·고유명사·수치 1자라도 변경되면 즉시 롤백.
- **장르 이탈 게이트**: 평어체 → 격식체 전환, 또는 그 반대 발생 시 롤백.

각 패턴별 수정 레시피는 위 S1 표의 "처방" 컬럼 + `~/.claude/skills/humanize-kr/.claude/skills/humanize-korean/references/quick-rules.md` 참조.

### 6단계: 자체검증 6항

| # | 항목 | 통과 기준 |
|---|---|---|
| 1 | 인용·고유명사·수치 보존 | 원문 대비 한 글자도 다르지 않음 |
| 2 | 변경률 ≤ 10% | 누적 변경 글자수 / 본문 글자수 |
| 3 | 장르·격식체 이탈 없음 | 입력 register 유지 |
| 4 | S1 잔존 0건 | 5단계 후 재검출 |
| 5 | 인공 비유 추가 없음 | 원문에 없던 수사 추가 금지 |
| 6 | 작가 voice 보존 | 핵심 비유 장치(예: 의인화) 절반 이상 유지 |

위반 시 해당 edit 롤백. 1회 재시도 후에도 실패하면 사용자에게 사람 검토 권고.

### 7단계: 리포트

산출물:
- `.context/humanize-post/{YYYY-MM-DD-NNN}/before.md` — 윤문 전 스냅샷
- `.context/humanize-post/{YYYY-MM-DD-NNN}/findings.json` — S1 finding 목록
- `.context/humanize-post/{YYYY-MM-DD-NNN}/diff.txt` — diff -u 결과
- `.context/humanize-post/{YYYY-MM-DD-NNN}/summary.md` — 등급, 변경률, 적용 내역

사용자에게 보고:

```
humanize-post 결과: {파일명}
- S1 finding: {N}건 ({카테고리 합계})
- 자동 윤문 적용: {M}건 (변경률 {X}%)
- 등급: {A/B/C/D}
- 산출물: .context/humanize-post/{run_id}/

{diff 요약 — 핵심 변경 3~5건만}
```

## 등급 기준 (자가 채점)

| 등급 | 조건 |
|---|---|
| A | S1 0건 (윤문 전 또는 후) |
| B | 윤문 후 S1 0건, 변경률 1~10% |
| C | S1 6건+ 발견되어 자동수정 중단 (정밀 윤문 권고) |
| D | 자체검증 6항 중 2개+ 위반 (사람 검토 필요) |

## 호출 예시

### write-essay에서 자동 호출 (직접)

```
write-essay 6단계 실행 →
  humanize-post 스킬을 작성한 파일 경로 인자로 호출:
  
  "humanize-post 스킬로 packages/blog/content/posts/{폴더명}/index.md를
   자체검증하고 S1 핀포인트 윤문 적용해줘. 결과를 결과 보고에 포함."
```

### 사용자 명시 호출

```
사용자: "방금 쓴 글 윤문해줘"
→ 1단계 (대상 파일 자동 결정 — 가장 최근 modified)
→ 2~7단계 진행
```

### 글로벌 정밀 윤문이 필요할 때

S1 6건+ 또는 사용자가 변경률 10% 초과 윤문을 원할 때:

```
권고 메시지:
"S1 패턴 {N}건 검출. 본 스킬은 자동수정 보류.
 정밀 윤문은 `cd ~/.claude/skills/humanize-kr && claude` →
 '이 글 자연스럽게 윤문해줘'로 별도 세션에서 진행 권장."
```

## 주의사항

- **인용 보호 절대원칙**: 큰따옴표 안 텍스트는 한 글자도 건드리지 않는다. 의심스러우면 보호.
- **voice 보존**: 글의 핵심 비유 장치(예: 어떤 추상 개념을 의인화한 비유)는 D-5 의인화 패턴이라도 절반 이상 보존. 본 스킬 S1 자동수정 대상에는 D-5를 포함하지 않는다 — 보존 우선.
- **draft 글 처리**: draft: true 글에도 동일하게 적용. 발행 여부와 무관.
- **재실행 안전성**: 같은 글에 재실행 시 S1 잔존 0건이면 SKIP. 무한 윤문 방지.
- **대상 파일 범위**: 블로그 포스트(`packages/blog/content/posts/**/index.md`)와 책 챕터(`_workspace/book/chapters/ch-*.md`). 챕터는 frontmatter가 없고 헤딩 규칙(`#`/`##`)이 다르지만 본문이 한국어 산문이라 동일 적용. 코드/README/스킬 정의 등 비-산문 파일은 대상 외 — 호출 시 거부.
