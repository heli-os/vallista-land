# 편집 리포트

## 전체 요약

원고의 전반적 완성도가 높다. Blueprint가 설계한 3막 구조(왜 작은 팀인가 → 작은 팀을 작동시키는 기술 → 작은 팀의 미래)가 원고 전체에서 자연스럽게 체감되며, 챕터 간 전환이 매끄럽다. 각 챕터가 고유한 메타포(UN 총회, 건강검진/연기 감지기, 워프 존 등)를 활용하여 중복을 피하면서도 책 전체의 톤 일관성을 유지하고 있다.

**강점:**
- 3막 서사 아크가 명확하게 작동함. 1막에서 제기한 "왜 작은 팀인가"라는 질문이 2막의 운영 기술을 거쳐 3막의 구조적 필연으로 수렴하는 흐름이 탄탄
- 챕터 간 전환 문장이 자연스러움. 각 챕터 마무리에서 다음 챕터의 핵심 질문을 유기적으로 제시
- 톤 가이드("~다" 체 + 탐구적 톤)가 14개 챕터 전체에서 일관되게 유지됨
- 블로그 포스트의 단순 요약이 아닌, 포스트 간 새로운 연결과 심화된 논증이 충분히 존재
- 프롤로그와 에필로그의 수미상관(아메리카노 두 잔 장면)이 효과적

**개선 사항:**
- ch-00에서 "랜딩"/"렌딩" 표기 불일치 발견 → 수정 완료
- ch-12에서 ch-01의 "바퀴" 메타포를 그대로 재사용 → 다른 비유로 교체 완료

## 4축 편집 상세 분석

### 일관성 (Consistency)
- **용어 통일**: 전체적으로 양호. "인재밀도", "What/How", "능력 범위", "Mandate Levels" 등 핵심 용어가 챕터 간 일관되게 사용됨
- **톤 통일**: "~다" 체가 전 챕터에서 일관. 독자에게 직접 말을 거는 부분은 "~인가?" 질문 형태로 Blueprint 톤 가이드에 부합
- **시제 일관**: 과거 경험 서술(과거형)과 원칙 제시(현재형)가 자연스럽게 구분됨
- **유일한 이슈**: ch-00 "렌딩/랜딩" 불일치 → 수정 완료

### 흐름 (Flow)
- **챕터 간 전환**: 매우 우수. 각 챕터 마지막 단락에서 다음 챕터의 핵심 질문을 자연스럽게 제기. 특히 2막(ch-03~09) 내부의 흐름이 채용→리더십→명료함→피드백→공정함→실행→번아웃으로 논리적 연쇄를 이룸
- **3막 구조 체감**: 1막(ch-00~02)에서 "왜"를 제기하고, 2막(ch-03~09)에서 "어떻게"를 탐구하며, 3막(ch-10~13)에서 "무엇을 향해"로 수렴하는 구조가 명확
- **내적 논리**: 앞뒤 챕터 간 모순 없음. ch-08의 "How 비용 0 수렴" 논지가 ch-11에서 자연스럽게 확장됨
- **페이스**: 프롤로그/에필로그(~3,000자)와 본문 챕터(4,000~6,000자)의 분량 배분이 적절하여 독자 피로가 잘 관리됨

### 깊이 (Depth)
- **표면적 요약 방지**: 블로그 포스트의 단순 나열이 아닌, 챕터 고유의 논증 구조와 새로운 연결이 존재. 예: ch-05에서 렌시오니의 여섯 가지 질문을 "여섯 개의 문, 하나의 복도" 메타포로 재구성하여 원 소스를 넘어서는 깊이 확보
- **새로운 연결**: ch-10에서 2막의 팀 운영 원칙이 제품 철학으로 자연스럽게 연결되는 점, ch-11에서 AI가 작은 팀의 강점을 "곱셈"으로 증폭시킨다는 구조적 논증이 독창적
- **구체적 사례**: 볼타의 실제 경험(3500:3 채용, 고객 문의 → 프로토타입 등)이 추상적 주장을 효과적으로 뒷받침

### 중복 (Redundancy)
- **축구 메타포**: ch-04(개인 기록 vs 팀 점수)와 ch-06(감독 vs 선수)에서 모두 사용하나, 관점이 명확히 다르며 Blueprint에서도 이 차별화를 의도한 것으로 확인. 허용 범위
- **바퀴 메타포**: ch-01(중심 비유)과 ch-12(부수적 비유)에서 중복 → ch-12에서 교체 완료
- **Company of One 인용**: ch-01(피하는 전략 vs 선택하는 전략)과 ch-12(철학 → 생존 전략)에서 다른 각도로 사용. Blueprint 설계 의도에 부합. 허용 범위
- **"How 비용 0 수렴"**: ch-08(운영 구조 관점), ch-11(팀 형태 관점), ch-12(회수/수렴)에서 각각 다른 층위로 활용. 3막 구조의 핵심 논지이므로 반복이 아닌 심화에 해당

## 결론

전체 14개 챕터 중 MAJOR 반려 없음. MINOR 2건을 직접 수정 완료. 모든 챕터가 PASS/MINOR(수정 완료) 상태이므로 **통합본(manuscript.md) 생성 가능**.
