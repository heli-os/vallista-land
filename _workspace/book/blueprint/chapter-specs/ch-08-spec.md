# 챕터 08: What과 How 사이에서

## 서사 아크 내 위치
제2막의 여섯 번째 챕터. Ch.03~07이 "사람"에 관한 운영 기술이었다면, Ch.08~09는 "일"에 관한 운영 기술이다. Ch.08은 2막의 클라이맥스로, 실행의 구조를 근본적으로 재설계하는 챕터.

## 핵심 논지
AI와 자동화 도구로 How(어떻게 만들 것인가)의 비용은 0에 수렴하지만, What(무엇을 만들 것인가)의 비용은 전혀 줄지 않았다. 이 비대칭이 팀을 느리게 만드는 역설을 낳는다. 해법은 What을 특정 직군만의 영역으로 두지 않고, 엔지니어를 포함한 팀 전체가 참여하는 것. 그리고 What과 How를 같은 스프린트에서 하지 않는 1/3 법칙으로 운영 구조를 바꾸는 것.

## 소스 포스트
- **빨라진 실행, 느려진 팀 -- 생산성의 역설** -> 사용할 부분:
  - "스프린트가 끝났다. 개발자들은 할 일이 없다" 오프닝
  - How 비용 0 수렴 논거 (시니어 2명 2주 -> 주니어 1명 3일)
  - What 비용은 변하지 않았다는 역설
  - 기업의 두 가지 선택지 (병목 충원 vs 개발자 감축) -- b가 압도적으로 쉬운 이유
  - What을 나눠야 한다: 고객 문의 기반, 벤치마크 세션, 프로토타이핑
  - 1/3 법칙: What과 How를 교차 운영
  - 직관 훈련: 외부 고객, 내부 고객, 시장, 예상치 못한 사용 관찰
- **볼타 팀은 이렇게 일합니다** -> 사용할 부분:
  - 직관 훈련의 4가지 방법 (외부 고객, 내부 고객, 시장, 제품 이해도)
  - 직관이 가능하게 하는 초 단위 의사결정
  - 탑다운 사고: How가 아닌 What과 Why에 집중
  - 빠른 검증과 용기 있는 실행

## 구성

### 도입: 스프린트가 끝났다, 개발자들은 할 일이 없다
"스프린트가 끝났다. 개발자들은 할 일이 없다. 기획은 아직 안 나왔고, 디자인은 리뷰 중이다." 팀의 실행 속도는 분명히 빨라졌는데, 팀 전체는 오히려 느려진 느낌. 이 역설의 구조를 파헤친다.

### 전개 1: How는 공짜가 되어가는 세상
AI 코딩 도구, 자동화 파이프라인으로 "만드는 비용"이 극적으로 줄었다. 하지만 "무엇을 만들 것인가"를 결정하는 비용은 전혀 줄지 않았다. 이 비대칭이 만드는 구조적 문제: PM/디자이너가 병목이 됨 -> 기업의 선택 a(병목 충원) vs b(개발자 감축) -> 왜 b가 현실에서 압도적으로 쉬운가. How만 잘하는 사람은 대체 가능해진다.

### 전개 2: What에 참여하는 엔지니어, 직관을 훈련하는 팀
해법 1: What을 나눈다. 고객 문의 기반 문제 파악, 벤치마크 세션, 프로토타이핑. 해법 2: 1/3 법칙으로 운영 구조를 바꾼다 (S1: 문제A 파악 + 아이디에이션 / S2: 문제A 실행 + 문제B 파악 / S3: ...). 이 구조의 핵심: What이 실패해도 How의 리소스가 날아가지 않는다.

볼타의 실전: 직관 훈련 4가지. 데이터보다 직관을 우선시하는 이유: 데이터 분석은 누구나 할 수 있지만, 데이터 없는 상황에서 판단하는 것은 훈련된 직관만 할 수 있다. 탑다운 사고로 목표에서 역산하는 방식.

### 마무리: 더 잘 정의하는 팀
"빨라진 실행을 위협으로 볼 수도 있고, 기회로 볼 수도 있다." 줄어든 실행 시간만큼 What에 참여할 여유가 생겼다. 살아남는 팀은 "더 빨리 만드는 팀"이 아니라 "더 잘 정의하는 팀"이다.

## 전환 지침

- "볼타 팀은 이렇게 일합니다" 포스트의 직관 훈련 4가지는 이 챕터에서 핵심적으로 활용. 에필로그에서 다시 다룰 때는 요약 수준으로만.
- 1/3 법칙의 도식(S1, S2, S3)을 시각적으로 제시하면 이해도가 높아진다. 집필 시 다이어그램 포함 권장.
- "신입 PM은 없다"는 표현이 PM을 폄하하는 것으로 읽히지 않도록 맥락 보강 필요.
