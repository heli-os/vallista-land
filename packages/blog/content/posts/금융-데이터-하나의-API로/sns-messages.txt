## 짧고 임팩트 있는 버전 (카카오톡/트위터)

78개 금융기관 데이터를 API 하나로. 3~6개월 걸리던 금융 데이터 연동, 이제 5분이면 됩니다. 수십억 건의 거래내역을 정규화해온 팀이 만든 Headless Finance Platform을 공개합니다.

## 스토리텔링 버전 (링크드인/페이스북)

은행마다 인증이 다르고, 카드사마다 응답 포맷이 다르고, 홈택스는 자기만의 세계. 금융 데이터 연동에 3~6개월을 쏟고도 새벽 장애에 시달리는 팀을 너무 많이 봐왔습니다. 그래서 만들었습니다. 78개 금융기관, 하나의 API, 12개 표준 스키마 — Headless Finance Platform. 더 이상 인프라에 시간을 뺏기지 않고, 제품에만 집중할 수 있도록.

## 개발자 커뮤니티용 (디스코드/슬랙/GeekNews)

금융 데이터 연동의 복잡성을 추상화한 Headless Finance Platform을 공개합니다. 78개 기관을 메타데이터 드리븐 아키텍처로 통합하고, 5개의 범용 핸들러가 모든 요청을 처리합니다. Node.js/Python/Java/Kotlin/Go SDK 지원, 동기·비동기·MQ 연동 모두 가능. 현재 Invite Only — h6s.ai

## 초간단 한 줄 버전 (카톡 단톡방/DM)

금융 데이터 연동 3~6개월 → API 하나로 5분. 볼타에서 만든 Headless Finance Platform 공개했어요 👇

## 링크드인 버전

"그 데이터 좀 볼 수 있게 해주세요"

이 한마디에 개발팀의 3개월이 날아갑니다.

—

카드 매출 불러오고, 은행 잔액 연동하고, 홈택스 세금계산서 붙이고.
단순해 보이지만, 막상 시작하면 —

은행마다 인증 방식이 다르고,
카드사마다 응답 포맷이 다르고,
홈택스는 자기만의 세계를 구축하고 있습니다.

겨우 연동에 성공해도,
기관 인터페이스가 예고 없이 바뀌면 새벽에 장애 알림이 울립니다.

하고 싶은 건 데이터 활용인데,
시간은 전부 연동에 빠집니다.

—

볼타를 운영하면서 똑같은 늪에 빠졌습니다.
수십억 건의 비정형 거래내역을 정규화하면서 깨달았습니다.
이 인프라를 모든 팀이 매번 새로 만드는 건 낭비라는 것을.

그래서 만들었습니다.
H6S — Headless Finance Platform.

—

Connect → Collect → Normalize.

1. 기관별로 천차만별인 인증 방식을, 하나의 인터페이스로 연결
2. API 한 줄로 데이터 수집. 대용량은 자동 청킹, 실패 시 자동 재시도
3. 어떤 기관의 데이터든 동일한 표준 스키마로 변환. AI가 가맹점명과 카테고리를 자동 매핑

—

78개 금융기관. 31개 데이터 액션. 12개 표준 스키마.
새 기관 추가 시 코드 변경 제로 — 메타데이터만으로 즉시 지원.
모든 민감 데이터는 AES-256 암호화, Live/Sandbox 환경 완전 분리.

—

연동 방식도 강제하지 않습니다.

동기 HTTP로 즉시 응답을 받을 수도,
비동기로 폴링하거나 Webhook 콜백을 받을 수도,
SQS나 Kafka 같은 메시지 큐로 연동할 수도 있습니다.

시스템 아키텍처에 맞는 인터페이스를 고르세요.
플랫폼이 거기에 맞춰드립니다.

—

그리고 하나 더.

12개의 표준 스키마, 일관된 엔드포인트, 예측 가능한 응답 구조.
이건 개발자의 인지 부하를 줄이기 위해 만든 것이지만,
AI 에이전트가 별도의 학습 없이 바로 연동할 수 있는 조건이기도 합니다.
