새벽 3시 장애 알림, 엔지니어 3명, 매달 5천만 원.
전부 금융 데이터 연동에 묶여 있었습니다.

—

사전 신청을 공개한 지 2시간.
70개 팀이 신청했습니다. 솔직히 예상하지 못한 속도였습니다.

그만큼 많은 팀이 같은 문제를 안고 있다는 뜻이기도 했습니다.
은행마다 다른 인증, 카드사마다 다른 포맷, 홈택스의 독자적인 세계. 겨우 연동해놔도 기관 인터페이스가 바뀌면 새벽에 장애 알림이 울립니다. 하고 싶은 건 데이터 활용인데, 시간과 비용은 전부 연동에 빠지는 구조.

이건 우리만의 문제가 아니었습니다.

—

내일(4월 17일), 볼타는 법인 설립 3년을 맞습니다.
1,066일.

그 시간 동안 23,199개의 고객사와 함께하며, 78개 금융기관에서 수십억 건의 거래내역을 수집하고, 수백만 건의 세금계산서를 처리하고, 서비스에 연결해왔습니다. 월 매출 성장률 20%를 이어오는 동안, 기관마다 다른 인증, 다른 포맷, 다른 장애 패턴을 전부 직접 마주했습니다. API 문서에 적혀 있지 않은 것들을 하나하나 몸으로 익힌 시간이었습니다.

그리고 확신했습니다.
이 인프라를 모든 팀이 매번 새로 만드는 건 낭비라는 것을.

—

최근 한 핀테크 스타트업의 이야기입니다.

금융 데이터 연동에 4개월을 쏟았습니다. 은행 3곳, 카드사 2곳, 홈택스. 기관마다 인증 방식이 다르고, 응답 포맷이 다르고, 장애 패턴이 다릅니다. 매달 5천만 원 이상의 인건비와 운영 비용이 연동 유지보수에 묶여 있었습니다. 인증서 관리, 기관별 예외 처리, 새벽 장애 대응. 엔지니어 3명이 풀타임으로 매달려 있었습니다. 정작 제품을 만들어야 할 사람들이, 인프라에 발목을 잡힌 상황이었습니다.

헤드리스를 도입한 뒤, 연동에 걸린 시간은 이틀.
엔지니어 3명은 본래 해야 할 제품 개발로 돌아갔습니다.
새벽 장애 대응은 더 이상 그 팀의 일이 아니게 됐습니다.
이제는 클로드 같은 AI에게 "우리 회사 3월 매출 분석해줘"라고 말하면, 헤드리스가 데이터를 연결합니다.

—

이번 헤드리스는 100개 기업만 우선 진행합니다.

금융 데이터는 팀마다 쓰는 기관이 다르고, 아키텍처가 다르고, 보안 요건이 다릅니다. 우리가 보장하려는 건 단순한 API 접근이 아니라, 각 팀의 환경에서 실제로 동작하는 수준의 안정성입니다. 그 기준을 지키려면 동시에 수백 팀을 받을 수 없습니다.

현재 내부 기준에 따라 순차적으로 연락드리고 있습니다.
신청하신 분들은 조금만 기다려 주시면 감사하겠습니다.
